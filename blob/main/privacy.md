# Privacy Policy

_Last updated: February 11, 2026_

This Privacy Policy describes how the software project, website, browser extensions, APIs, and related services (collectively, the **“Service”**) collect, use, and protect information.

The Service is designed with a strong emphasis on **data minimization, transparency, and security**. This document explains what information may be processed, how it is used, and what choices users have regarding their data.

By using the Service, you agree to the practices described in this Privacy Policy.

---

# 1. Scope

This Privacy Policy applies to all components of the Service, including:

- public websites and documentation
- browser extensions
- APIs and backend services
- developer tools and integrations
- related applications operated by the project

If a specific feature requires additional data handling practices, those details may be described in feature-specific documentation.

---

# 2. Design Principles

The Service follows several guiding principles when handling data:

**Minimal Collection**  
Only the information necessary to operate the Service is collected.

**Security First**  
Infrastructure and services are configured with modern security practices.

**Transparency**  
Users should always understand what data is processed and why.

**Operational Necessity**  
Data is processed only for legitimate operational purposes such as reliability, security, or requested functionality.

---

# 3. Information We May Process

Depending on how the Service is used, different types of information may be processed.

## 3.1 Technical and Diagnostic Information

When interacting with the Service, certain technical metadata may be automatically generated. This may include:

- browser type and version
- operating system
- device type
- timestamps of requests
- service response times
- error logs and diagnostic events
- API request metadata

This information is used exclusively for:

- debugging system failures
- monitoring service reliability
- preventing abuse or malicious activity
- improving system performance

These logs are intended to contain **technical metadata only** and are not designed to include personal information.

---

## 3.2 Integration and Configuration Data

If the Service integrates with third-party platforms, limited integration metadata may be stored.

Examples include:

- integration identifiers (such as shop domains or workspace IDs)
- installation timestamps
- configuration settings
- API tokens or credentials required for integration
- webhook identifiers or event metadata

This information is required to maintain operational integrations with external systems.

---

## 3.3 User-Provided Content

Users may voluntarily submit information when using the Service.

Examples include:

- prompts or input data
- configuration parameters
- API requests
- uploaded files
- commands issued through the interface

This data is processed only for the purpose of executing the requested operation.

# 4. AI Processing

Certain features of the Service may use artificial intelligence or machine learning systems.

When AI-powered features are used:

- input data may be transmitted to AI service providers
- those providers process the input to generate outputs
- responses are returned to the Service and presented to the user

AI requests may be logged for operational debugging and quality improvements.

The Service attempts to minimize unnecessary transmission of personal information to AI systems.

# 5. Cookies and Local Storage

The Service may use local storage mechanisms such as:

- browser local storage
- session storage
- authentication tokens
- temporary application state

These mechanisms are used strictly for functionality such as:

- maintaining sessions
- storing configuration preferences
- improving user experience

Operational logs may be retained temporarily for reliability and debugging purposes.

Typical retention periods may include:

- short-term diagnostic logs
- temporary request metadata
- system performance metrics

Logs are periodically rotated or deleted as part of infrastructure maintenance.

Reasonable security measures are implemented to protect data processed by the Service.

These may include:

- encrypted connections (HTTPS / TLS)
- infrastructure-level access controls
- secure API authentication
- monitoring for abnormal activity
- regular dependency and security updates

No system can guarantee absolute security; however, the Service follows industry-standard practices to reduce risk.

The Service may rely on external infrastructure providers or APIs to operate.

Examples may include:

- cloud hosting providers
- AI model providers
- monitoring and logging services
- authentication services

# 9. Data Minimization

The Service is designed to avoid collecting unnecessary personal information.

Whenever possible:
